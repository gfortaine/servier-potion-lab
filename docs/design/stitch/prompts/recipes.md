# Stitch Prompt — Recipes

Design the recipes and created-potions route for SERVIER Potion Lab.

**Purpose:** Let a reviewer inspect discovered recipes, guide a recipe back to composer, and verify created potions.

**Platform:** Web page, responsive.

**Page structure:**

1. **Compact route heading:** title “Recettes & potions.” No dashboard hero and no inventory controls.
2. **Formula portfolio:** nine canonical recipe cards. Discovered cards reveal potion name and exact ingredient signature. Locked cards clearly indicate how to guide composition without hiding the existence of the recipe.
3. **Completion gauge:** progress from discovered recipes, using real app data only.
4. **Created-potion ledger:** chronological list of created potions, newest highlighted, ingredients shown as formula signatures.
5. **Success banner:** when returning from composer, show a non-blocking success confirmation and a link to compose again.

Use the design system from `DESIGN.md`. Keep the layout asymmetric and premium; avoid fake statistics and decorative clutter.
