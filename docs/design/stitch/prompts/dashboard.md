# Stitch Prompt — Dashboard

Design the dashboard for a desktop-first SERVIER technical-test app named **SERVIER Potion Lab**.

**Purpose:** Create a striking first viewport that invites the reviewer to compose a potion while proving this is a precise pharma discovery workflow.

**Platform:** Web, desktop-first, responsive down to mobile.

**Page structure:**

1. **Sticky SERVIER header:** real SERVIER logo, compact route navigation for Accueil, Composer, Recettes, Inventaire, active route state.
2. **Compact asymmetric hero theatre:** left-aligned headline “Atelier des potions SERVIER.”, concise body copy, one primary CTA to compose a potion, no secondary hero CTA.
3. **Discovery cockpit visual:** abstract SERVIER proximity motif, formula traces, three-socket protocol hint, and deterministic discovery framing.
4. **Launchpad:** three route cards for composer, recipes, and inventory with unequal widths or staggered placement, not a generic equal-card row.
5. **Discovery funnel:** show real domain facts only: 14 ingredients, 364 possible three-ingredient candidates, 9 canonical recipes. Mention that secret hypotheses are isolated and never mutate canonical recipes.

Use the design system from `DESIGN.md`. Do not invent metrics, credentials, fake performance claims, new ingredients, or extra recipes. Keep the hero compact enough that the launchpad is visible in the first viewport.
